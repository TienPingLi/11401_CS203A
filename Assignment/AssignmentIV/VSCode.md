

# VS Code 安裝與使用過程心得報告

在本次設定 VS Code 的過程中，我遇到了一些意料之外的問題。以下記錄我實際遇到的狀況、排除方式與後續心得，希望未來能成為自己或他人參考的經驗。

---

## 一、VS Code 同步 GitHub 時的權限問題

### ◆ 問題描述

在嘗試使用 VS Code 內建的 Git 功能進行版本同步時，系統多次跳出無權限（Permission denied）或認證失敗（Authentication failed）的訊息。即使重新登入 GitHub 帳號或重設 Token，都仍無法順利 push。

### ◆ 解決方式

後來改採 **GitHub Desktop** 處理 Commit 與 Push 的動作，發現登入過程更直覺，而且不會再出現權限拒絕的情況。使用 GitHub Desktop 後，所有推送都能順利完成，也讓版本控制流程穩定許多。

### ◆ 心得

VS Code 雖然整合了 Git 功能，但實際使用上仍可能受限於 Token、權限、或設定檔的細節。對我來說，GitHub Desktop 反而更適合作為入門的版本控制工具，能大幅降低設定成本。

---

## 二、VS Code 中無法執行 C++ 程式

### ◆ 問題描述

在 VS Code 裡設定 C++ 環境時，發現只有在一個獨立的 test.cpp 測試檔中能成功使用「執行與偵錯」印出 `hello world`。
然而，真正的專案主程式卻始終無法執行，不管怎麼修改 tasks.json 或 launch.json，都沒有成功過。

### ◆ 解決方式

後來我選擇改用 **Visual Studio 2022 (VS2022)** 進行 C++ 的編譯與偵錯。
VS2022 內建完整的 MSVC 工具鏈，不需要額外設定執行環境，一次就能成功編譯並順利執行主程式。

### ◆ 心得

VS Code 是一個輕量且彈性的 IDE，但對 C++ 來說仍需要額外設定 compiler、路徑與偵錯工具，對我而言不太友善。反而是 VS2022 的整合度較高，在做 C++ 專案時更省時省力。

---

## ◆ 總結

這次的環境設定經驗讓我理解到：

* VS Code 功能強大，但設定門檻高
* GitHub Desktop 更適合初學者進行版本控制
* VS2022 在 C++ 專案上更穩定可靠

雖然花了不少時間排除問題，但也因此更熟悉了不同工具之間的差異，希望未來能更有效率地使用開發環境。
